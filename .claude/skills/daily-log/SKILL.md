# トレーニング日報の作成

以下の手順でユーザーにヒアリングし、日報ファイルを作成する。

## 手順

### 1. 日付と計画の特定

- 引数 `$ARGUMENTS` が指定されていればその日付を使用。なければ今日の日付を使う
- `training_plans/` 内のサイクルのうち、該当日を含む計画の `plan.md` を特定する
- plan.md を読み、該当日が Week 何の何曜日か、予定メニューが何かを特定する
- 同じサイクルの `daily_log/` ディレクトリにある既存のテンプレート `template.md` を読む

### 2. ヒアリング（AskUserQuestion を使って一括で聞く）

以下を **2回の AskUserQuestion** に分けて聞く。

#### 質問1: コンディション + 練習概要

以下の項目をまとめて1つの AskUserQuestion で聞く（各項目を questions 配列の個別質問にする）:

1. **安静時心拍数**: 自由入力（"安静時心拍数(bpm)は？（Garmin等の値）"）
   - 選択肢: 「未測定」を含む + Other で自由入力
2. **起床時体重**: 自由入力（"起床時体重(kg)は？"）
   - 選択肢: 「未測定」を含む + Other で自由入力
3. **睡眠**: 自由入力（"睡眠時間と質は？"）
   - 選択肢: 「7h以上・良好」「6-7h・普通」「6h未満・不良」 + Other
4. **体調・疲労**: （"疲労感と痛み・違和感は？"）
   - 選択肢: 「疲労なし・痛みなし」「軽い疲労あり・痛みなし」「疲労あり・痛みあり（部位を記入）」 + Other
5. **練習を予定通り実施したか**: 予定メニューを提示した上で聞く
   - 選択肢: 「予定通り実施」「一部変更あり（詳細を記入）」「別メニューに変更」「休養にした」

#### 質問2: 練習の詳細

質問1の回答を踏まえて聞く（休養の場合はスキップ）:

1. **距離と時間**: 自由入力（"総距離(km)と所要時間は？"）
   - 選択肢: 距離の概算候補を計画から算出 + Other
2. **RPE（主観的運動強度）**: （"きつさは？"）
   - 選択肢: 「3-4 (楽)」「5-6 (ややきつい)」「7-8 (きつい)」「9-10 (非常にきつい)」
3. **補強トレーニング**: （"補強は実施した？"）
   - 選択肢: 「A: レジスタンス実施」「B: サーキット実施」「実施せず」
4. **メモ**: 自由入力（"気づきやメモがあれば"）
   - 選択肢: 「特になし」 + Other

### 3. ファイル生成

- テンプレートの構造に沿って、ヒアリング結果を埋めた日報を作成
- ファイルパス: 該当サイクルの `daily_log/YYYYMMDD.md`
- タイトルの `W? Day?` を実際の週番号と曜日で置換
- 予定メニューと実施メニューの両方を記載
- Q日でワークアウト詳細テーブルが不要な場合（E日・L日・休養日）は、そのセクションを省略する
- 起床時体重をコンディション欄に記入する

### 4. 完了報告

- 作成したファイルのパスを伝える
- コンディションに注意サインがあれば（心拍数異常、痛み、睡眠不足など）プランの故障予防ガイドラインに基づいてアドバイスする

## 注意事項

- 言語は日本語
- ヒアリングは簡潔に。ユーザーの負担を最小限にする
- Q日（月曜Q1・木曜Q2）の場合、ワークアウト詳細テーブルを含める
- E日・L日・休養日の場合、ワークアウト詳細テーブルは省略
- 該当サイクルの plan.md が見つからない場合はユーザーに確認する
